Este backlog apresenta a evolução do projeto e as próximas atividades planejadas, seguindo o método Scrum e as mudanças de requisitos do cliente.

Entregue (Final) Prioridade Máxima

Sprint Final: Autenticação, Mobile e Recursos Nativos

  • Autenticação: Implementação de Context API para login, logout e proteção de rotas privadas (Home, Criar, Favoritos).
  • Recurso Nativo: Integração com a câmera do dispositivo através da API MediaDevices para captura de fotos de eventos.
  • Mobile Optimization: Ajustes de UX para iOS (evitar zoom em inputs) e Android (tamanhos de toque).
  • Arquitetura: Migração para Context API para gerenciamento de estado global de usuário.
Entregue (Sprint 3) Prioridade Média

Sprint 3: Favoritos e Notificações

  • Contador reativo: Navbar com badge de favoritos em tempo real
  • Persistência: Sincronização com localStorage
  • UX: Feedback visual ao adicionar/remover itens
Entregue (Sprint 2) Prioridade Alta

Sprint 2: CRUD e Filtros

  • CRUD: Criação e listagem de eventos via interface
  • Filtros dinâmicos: Filtragem por localidade com "Todos"
  • LocalStorage: Salvamento persistente dos eventos criados
Sprint 5 (Atual) Prioridade Alta

Gestures, Navegação e Testes

  • Menu de navegação: Menu lateral (Drawer) para mobile
  • Gestures: Implementação de swipe lateral para favoritar/descartar cards
  • Qualidade: Cobertura de testes unitários com React Testing Library
  • Roteamento: Melhoria no fluxo de navegação entre Home, Criar e Favoritos
Futuro (Nativo) Estratégico

Transição para Mobile Nativo

  • Avaliação técnica para React Native
  • Sincronização offline-first
  • Push Notifications nativas
